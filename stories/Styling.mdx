`JBModal` styling is split between host variables, official shadow parts, slotted content, and the `open` custom state. Use variables for the core shell first, then use parts for backdrop, content box, and animation.

## Common Design Tokens

Use the core theme docs as the source of truth for shared design-system variables:

- [Theme Colors](../../jb-core/theme/stories/colors.mdx) explains the general color palette variables, default colors, customization approaches, and color generation helpers.
- [Theme Sizes](../../jb-core/theme/stories/sizes.mdx) explains shared size and radius variables.
- [Theme Custom Galleries](https://javadbat.github.io/design-system/?path=/docs/theme-custom-material--docs) gathers the shared custom style recipes across components.

Modal recipes should use those common tokens where possible, then map them into `--jb-modal-*` variables and official parts where the modal needs component-specific values.

## Styling Model

Use four layers when styling modal examples:

- Common design tokens are documented in [Theme Colors](../../jb-core/theme/stories/colors.mdx) and [Theme Sizes](../../jb-core/theme/stories/sizes.mdx).
- Reusable example theme packs live in [`docs/styles`](https://github.com/javadbat/design-system/tree/main/docs/styles) and can be shared across components.
- Component recipe files live in [`modules/jb-modal/stories/styles`](https://github.com/javadbat/design-system/tree/main/modules/jb-modal/stories/styles).
- Public component variables use `--jb-modal-*` on `jb-modal`; official parts use `::part(component-wrapper)`, `::part(background)`, and `::part(content-box)`.
- Trigger and action buttons reuse the existing [JBButton Styling](../../jb-button/stories/Styling.mdx) recipe classes instead of redefining button variables in modal CSS.

Each example uses one shared example theme pack from [`docs/styles`](https://github.com/javadbat/design-system/tree/main/docs/styles) and one modal-specific recipe from [`modules/jb-modal/stories/styles`](https://github.com/javadbat/design-system/tree/main/modules/jb-modal/stories/styles). The gallery uses `style-preview.css` only to render fixed modals as local previews inside the documentation grid.

## Recommended Customization Path

1. Use `--jb-modal-*` variables first for backdrop color, content background, content radius, and modal stack level.
2. Use `::part(background)` for backdrop blur, opacity, and entrance motion.
3. Use `::part(content-box)` for border, shadow, content-box sizing, and entrance motion.
4. Use the `open` custom state for motion selectors such as `jb-modal:state(open)::part(content-box)`.
5. Style header, content, and footer by styling the slotted elements passed to `slot="header"`, `slot="content"`, and `slot="footer"`. See [Slot Demo](./slot-demo.mdx).
6. Use [Animation](./animation.mdx) for deeper enter and close animation patterns.
7. Reuse [JBButton Styling](../../jb-button/stories/Styling.mdx) classes for modal trigger, cancel, and submit buttons.
8. Use common theme variables from [Theme Colors](../../jb-core/theme/stories/colors.mdx) and [Theme Sizes](../../jb-core/theme/stories/sizes.mdx) when the style should align with the design system.
9. Put reusable theme-pack values in [`docs/styles`](https://github.com/javadbat/design-system/tree/main/docs/styles) only when they should be shared by multiple component demos.
10. Add a new modal variable or official part only when a needed internal surface cannot be customized with current public variables or official parts.

## Starter Recipe

Use this minimal recipe when a product only needs brand alignment and simple motion.

```css
jb-modal.brand-modal {
  --jb-modal-bg-color: #ffffff;
  --jb-modal-back-bg-color: rgb(15 23 42 / 0.48);
  --jb-modal-border-radius: 1rem;
  --jb-modal-z-index: 1000;
}

jb-modal.brand-modal::part(background) {
  backdrop-filter: blur(0.5rem);
}

jb-modal.brand-modal::part(content-box) {
  border: 1px solid #dbeafe;
  box-shadow: 0 1.25rem 2.5rem rgb(15 23 42 / 0.16);
}

jb-modal.brand-modal:state(open)::part(content-box) {
  animation: brand-modal-in 180ms ease-out;
}

@keyframes brand-modal-in {
  from {
    opacity: 0;
    transform: translateY(0.75rem);
  }
}
```

## Theme File Pattern

Use [Theme Colors](../../jb-core/theme/stories/colors.mdx) and [Theme Sizes](../../jb-core/theme/stories/sizes.mdx) as the common design-token guides. Theme-pack files should keep reusable demo variables in [`docs/styles`](https://github.com/javadbat/design-system/tree/main/docs/styles), then each component should keep its own style CSS in its story [`styles`](https://github.com/javadbat/design-system/tree/main/modules/jb-modal/stories/styles) folder.

```css
/* https://github.com/javadbat/design-system/blob/main/docs/styles/carbon.css */
.theme-name-style {
  /* shared design-system variables */
}

/* https://github.com/javadbat/design-system/blob/main/modules/jb-modal/stories/styles/style-carbon.css */
jb-modal.theme-name-modal {
  /* modal host variables */
}

jb-modal.theme-name-modal::part(background) {
  /* backdrop treatment */
}

jb-modal.theme-name-modal::part(content-box) {
  /* modal surface treatment */
}

jb-modal.theme-name-modal:state(open)::part(content-box) {
  /* open-state motion */
}

jb-button.theme-name-button {
  /* imported from the JBButton recipe, not duplicated here */
}
```

This keeps common theme tokens reusable while making the `JBModal` styling API visible in one page.
